# UCID — Universal Concept ID

UCID is a universal, language‑independent system of **digital semantic atoms**.  
Each concept in the world — physical, abstract, cultural, scientific — receives a **unique numerical identifier** that is:

- unambiguous  
- universal  
- permanent  
- language‑agnostic  
- machine‑friendly  
- human‑interpretable  

UCID does **not** replace human languages.  
It replaces **ambiguity**, **translation errors**, **semantic drift**, and **textual noise**.

# Why UCID Is Needed

Human languages are **incompatible**.  
One word in one language may correspond to:

- a single concept  
- multiple concepts  
- dozens of concepts  
- or nothing at all in another language  

This creates ambiguity for humans and catastrophic errors for AI.

### Examples of linguistic chaos

| Language | Word | Problem |
|---------|------|---------|
| Russian | “лук” | plant / weapon |
| English | “set” | 430+ meanings |
| German | Schadenfreude | no direct equivalent |
| Japanese | Ikigai | requires a paragraph to explain |

### UCID solution  
Each meaning receives **one unique ID**:

- `01.04.233` — onion (plant)  
- `01.04.987` — bow (weapon)  
- `02.11.045` — Schadenfreude  
- `02.11.046` — Ikigai  

No ambiguity.  
No translation.  
No loss of meaning.

# UCID Format

Minimal, strict, extensible:

```
UCID = <Domain><Class><Atom><Version><Meta>
```

Example:

```
02.11.046.1.0
```

| Field | Meaning |
|-------|---------|
| Domain | physics / biology / culture / law / etc. |
| Class | subclass within domain |
| Atom | unique semantic atom |
| Version | revision |
| Meta | flags (abstract, contextual, entity type) |
